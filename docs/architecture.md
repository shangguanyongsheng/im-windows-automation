# 架构设计

## 整体架构

本项目采用三层架构设计：

```
+------------------+     +------------------+     +------------------+
|   IM 平台        |     |   指令解析 Agent  |     |  Windows-MCP     |
| (飞书/钉钉/微信/QQ)|<--->|                  |<--->|   客户端         |
+------------------+     +------------------+     +------------------+
```

## 组件说明

### 1. IM 适配器层

- **BaseIM**: 抽象基类，定义统一的 IM 接口
- **FeishuAdapter**: 飞书平台适配器
- **DingTalkAdapter**: 钉钉平台适配器
- **WeChatAdapter**: 微信平台适配器
- **QQAdapter**: QQ 平台适配器

每个适配器负责处理特定 IM 平台的消息接收、发送和认证。

### 2. 指令解析 Agent 层

- **CommandAgent**: 负责解析用户自然语言指令
- 使用 NLP 技术识别操作意图和参数
- 将自然语言转换为结构化的命令对象

### 3. Windows-MCP 客户端层

- **WindowsMCPClient**: 与 Windows-MCP 服务器通信
- 封装 MCP 协议调用
- 处理请求和响应

## 数据流

1. 用户在 IM 平台发送消息
2. 对应的 IM 适配器接收消息并转发给 CommandAgent
3. CommandAgent 解析指令，生成结构化命令
4. WindowsMCPClient 执行命令并获取结果
5. 结果通过 IM 适配器返回给用户

## 扩展性

- 新增 IM 平台只需实现 BaseIM 接口
- 新增 Windows 操作只需在 Windows-MCP 服务器端添加
- 指令解析可通过训练模型不断优化
