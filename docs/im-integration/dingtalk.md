# 钉钉集成指南

## 准备工作

1. **创建钉钉企业内部应用**
   - 登录 [钉钉开放平台](https://open.dingtalk.com/)
   - 进入「应用开发」→「企业内部开发」→「机器人」
   - 点击「创建机器人」

2. **配置机器人基本信息**
   - 机器人名称: `Windows 自动化助手`
   - 机器人头像: 选择合适的图标
   - 安全设置: 建议使用加签方式

3. **获取必要凭证**
   - 保存 `AppKey` 和 `AppSecret`
   - 如果使用加签，保存签名密钥

## 配置环境变量

在项目根目录创建 `.env` 文件:

```env
DINGTALK_APP_KEY=your_app_key
DINGTALK_APP_SECRET=your_app_secret
DINGTALK_SIGN_SECRET=your_sign_secret  # 如果使用加签
```

## 实现消息处理

钉钉机器人支持以下交互方式:

1. **Webhook 方式**: 直接向机器人 Webhook URL 发送消息
2. **事件订阅**: 接收用户发送的消息事件

推荐使用事件订阅方式以支持双向交互。

## 安全注意事项

- 使用 HTTPS 确保通信安全
- 验证钉钉服务器签名防止伪造请求
- 不要在代码中硬编码敏感信息
- 限制机器人可见范围

## 测试

1. 在钉钉群聊中添加机器人
2. 向机器人发送测试消息
3. 检查日志确认消息接收和处理正常

## 故障排除

- **签名验证失败**: 检查时间戳和签名算法
- **认证失败**: 检查 AppKey 和 AppSecret
- **无法接收消息**: 确认事件订阅配置正确
- **消息发送失败**: 检查机器人是否在目标群聊中
